0전체 흐름 미리 보기

Flutter 개발을 시작하려면 아래 항목이 필요합니다. 각 단계가 왜 필요한지 먼저 짚고 시작하면 헷갈리지 않습니다.

  • JDK 21 — Android 앱을 빌드하기 위한 Java 실행 환경. (Android Studio에 JBR로 내장돼 있지만, 명령줄에서 빌드할 때를 대비해 별도 설치 권장)
  • Android Studio — 코드를 작성하고 디버깅하는 메인 IDE이자 Android SDK · 에뮬레이터 관리 도구.
  • Flutter SDK — Flutter의 본체. flutter 명령어가 여기서 나옵니다.
  • Android SDK + 에뮬레이터 + 라이센스 동의 — Android 앱을 빌드 · 실행하기 위한 도구와 동의 절차.
  • Git (선택) — flutter upgrade 같은 SDK 업데이트 · 채널 전환 시에만 필요. 일상적인 개발(flutter run, flutter build)에는 없어도 됩니다.
💡 Tip. 설치 도중 막히면 맨 아래 트러블슈팅을 먼저 보세요. 거의 모든 초보자가 같은 곳에서 막힙니다.

1JDK 21 설치

Android Gradle Plugin이 JDK 21을 요구합니다. 가장 무난한 선택은 Eclipse Temurin (Adoptium)의 JDK 21 LTS입니다.

다운로드

  1. adoptium.net의 Temurin 21 다운로드 페이지에 접속합니다.
  2. OS는 Windows, 아키텍처는 x64, 패키지 형식은 .msi를 선택합니다.
  3. 설치 마법사 실행 → Set JAVA_HOME variable과 Add to PATH 옵션을 반드시 체크하고 설치합니다.

설치 확인

새 터미널(PowerShell)을 열고 아래를 입력합니다.

PS> java -version
openjdk version "21.0.x" ...

버전이 21로 나오면 성공입니다.

📌 알아두기. 최신 Android Studio는 내부에 JBR(JetBrains Runtime, JDK 21 기반)을 포함하고 있어서 IDE 안에서만 빌드하면 별도 JDK가 없어도 동작합니다. 하지만 명령줄로 flutter build를 돌리려면 시스템 JDK가 필요하므로 그냥 설치해두는 게 편합니다.

2Android Studio 설치

Flutter 개발의 중심 IDE입니다. Android SDK · 에뮬레이터 관리도 모두 여기서 합니다.

  1. developer.android.com/studio에서 최신 안정 버전(Stable)을 다운로드합니다.
  2. 설치 마법사 진행 시 옵션은 모두 기본값(Next)으로 둡니다.
  3. 설치 후 처음 실행하면 Setup Wizard가 뜹니다. Standard를 선택해 진행하세요. 그러면 권장 Android SDK · 플랫폼 도구가 자동으로 함께 설치됩니다.

Flutter · Dart 플러그인 추가

  1. Android Studio 첫 화면에서 More Actions > Plugins를 엽니다. (또는 File > Settings > Plugins)
  2. 검색창에 Flutter를 입력하고 설치합니다. (Dart 플러그인은 자동으로 같이 깔립니다.)
  3. 설치 후 Android Studio를 한 번 재시작합니다.

3Android SDK와 도구 설치

Android Studio 시작 화면에서 More Actions > SDK Manager를 클릭합니다. (프로젝트가 열려 있다면 Tools > SDK Manager)

SDK Platforms 탭

  • 가장 최신 API Level(현재 API 36)을 체크합니다.
  • 오른쪽 아래 Apply를 누른 뒤 라이센스 약관에 동의하고 설치합니다.

SDK Tools 탭

다음 항목이 모두 체크돼 있는지 확인하고, 빠진 게 있으면 체크 후 Apply를 누릅니다.

  • Android SDK Build-Tools
  • Android SDK Command-line Tools
  • Android Emulator
  • Android SDK Platform-Tools
  • CMake
  • NDK (Side by side)
⚠️ 중요. Android SDK Command-line Tools를 꼭 설치해야 합니다. 이게 없으면 다음 단계의 flutter doctor --android-licenses 명령이 실패합니다.

4Flutter SDK 설치

Flutter SDK는 압축 파일을 풀어서 적당한 폴더에 두기만 하면 됩니다. 경로에 한글이나 공백, 특수문자가 들어가면 안 됩니다.

다운로드 & 압축 풀기

  1. docs.flutter.dev/install/archive의 Stable channel (Windows)에서 최신 zip을 받습니다.
  2. 다음 위치에 압축을 풉니다 (사용자 폴더 아래 권장):
    권장 경로 C:\src\flutter
    피해야 할 경로 C:\Program Files\... (관리자 권한 필요), C:\Users\홍길동\... (한글 포함)
  3. 압축을 풀면 C:\src\flutter\bin 폴더가 생깁니다. 이 경로를 다음 단계에서 PATH에 등록합니다.

5환경변수 PATH에 Flutter 등록

이 단계를 거쳐야 어디서든 터미널에서 flutter 명령을 사용할 수 있습니다.

  1. Windows 검색에 "환경 변수"를 입력 → 시스템 환경 변수 편집 클릭.
  2. 환경 변수(N)... 버튼 클릭.
  3. 위쪽 사용자 변수 영역에서 Path를 선택 → 편집.
  4. 새로 만들기를 누르고 다음 경로를 추가: 
    C:\src\flutter\bin
  5. 확인을 눌러 모든 창을 닫고, 이미 열려 있던 모든 터미널을 종료한 뒤 새 터미널을 엽니다. (이게 핵심!)

확인

PS> flutter --version
Flutter 3.x.x • channel stable ...

버전 정보가 나오면 성공입니다.

6Android 라이센스 동의

Android SDK는 사용 전에 라이센스에 동의해야 합니다. 이걸 안 하면 빌드할 때 오류가 납니다.

PS> flutter doctor --android-licenses

화면에 약관이 한 묶음씩 나타납니다. 매번 y를 누르고 Enter를 치며 모두 동의합니다. 마지막에 다음 메시지가 보이면 끝:

All SDK package licenses accepted.
⚠️ "cmdline-tools component is missing" 에러가 난다면? Step 3의 SDK Tools 탭에서 Android SDK Command-line Tools를 설치하지 않은 것입니다. 다시 돌아가 설치하세요.

7Android 에뮬레이터 만들기

실제 휴대폰을 USB로 연결해도 되지만, 처음에는 PC 안에서 도는 에뮬레이터가 가장 편합니다.

VM 가속 활성화 (Windows)

BIOS에서 Intel VT-x 또는 AMD-V(SVM)가 켜져 있어야 합니다. 보통 최신 PC는 기본으로 켜져 있습니다.

에뮬레이터(AVD) 생성

  1. Android Studio 시작 화면에서 More Actions > Virtual Device Manager를 엽니다. (또는 Tools > Device Manager)
  2. 왼쪽 위 + Create Virtual Device 클릭.
  3. Form Factor에서 Phone 선택 → 예: Pixel 8 선택 후 Next.
  4. System Image: 사용 중인 CPU에 따라 x86 Images(Intel/AMD) 또는 ARM Images(Snapdragon X 등) 탭에서 최신 안정 Android 버전을 선택합니다. 이미지 옆 ⬇️ 아이콘이 있으면 클릭해 먼저 다운로드.
  5. Additional settings → Emulated Performance → Graphics: Hardware로 설정합니다.
  6. Finish를 누르면 가상 기기가 생성됩니다.
  7. Device Manager에서 만든 기기 옆 ▶ 버튼을 눌러 한 번 부팅해봅니다. 잠시 뒤 화면에 안드로이드 홈이 뜨면 성공입니다.

8flutter doctor로 최종 점검

모든 설치를 한 번에 점검해주는 만능 명령어입니다.

PS> flutter doctor

이상적인 결과는 다음과 같습니다. 모든 항목 앞에 ✓ 가 떠야 합니다.

[✓] Flutter (Channel stable, 3.x.x, on Microsoft Windows ...) [✓] Windows Version (Installed version of Windows ...) [✓] Android toolchain - develop for Android devices (Android SDK version 36.x.x) [✓] Chrome - develop for the web [✓] Visual Studio - develop Windows apps (optional) [✓] Android Studio (version 202x.x) [✓] VS Code (optional) [✓] Connected device (1 available) [✓] Network resources • No issues found!

만약 ✗ 나 ! 가 보이면, 메시지에 적힌 해결책을 그대로 따라 하면 됩니다.

9첫 Flutter 앱 만들고 실행

프로젝트 생성

  1. Android Studio 시작 화면에서 New Flutter Project 클릭.
  2. 왼쪽에서 Flutter 선택 → Flutter SDK path에 C:\src\flutter 입력 → Next.
  3. Project name: 소문자/언더바만 사용 (예: my_first_app).
  4. Project location은 한글이 없는 경로로. (예: C:\src\my_first_app)
  5. Create 클릭. SDK 다운로드와 첫 빌드까지 수 분 걸릴 수 있습니다.

실행

  1. 상단 디바이스 선택 메뉴에서 Step 7에서 만든 에뮬레이터를 선택합니다.
  2. ▶ 실행 버튼(또는 Shift+F10)을 누릅니다.
  3. 에뮬레이터에 카운터 앱이 뜨고, 오른쪽 아래 + 버튼을 누르면 숫자가 올라갑니다. 🎉

Hot Reload 체험

lib/main.dart에서 'You have pushed the button this many times:' 문구를 자유롭게 바꿔보세요. 저장(Ctrl+S)하는 순간 에뮬레이터 화면이 즉시 갱신됩니다. 이것이 Flutter의 가장 강력한 기능 중 하나, Stateful Hot Reload입니다.

+(선택) Git 설치 — SDK 업데이트 시에만 필요

Git이 없어도 위 9단계까지는 모두 동작하고 앱도 정상적으로 빌드 · 실행됩니다. Git이 실제로 필요해지는 시점은 다음과 같습니다.

  • flutter upgrade — Flutter SDK를 최신 버전으로 올릴 때
  • flutter downgrade — 이전 버전으로 되돌릴 때
  • flutter channel stable / beta / master — 릴리스 채널을 전환할 때

이 명령들은 모두 내부적으로 git pull을 호출하기 때문에, Git이 없으면 위 명령만 실패합니다. 일상 개발에는 영향 없습니다.

설치 방법

  1. git-scm.com/downloads/win에서 64-bit Git for Windows Setup을 받습니다.
  2. 설치 마법사는 옵션을 바꾸지 말고 모두 기본값(Next)으로 진행합니다.
  3. 새 터미널에서 git --version으로 확인.
💡 참고. flutter doctor는 Git 설치 여부를 별도 항목으로 표시하지 않습니다. SDK 업데이트가 필요할 때 비로소 에러 메시지로 알려주는 식입니다.

⚠자주 만나는 오류 해결

1. flutter 명령을 찾을 수 없다고 나옵니다.

PATH 등록 후에는 열려 있던 터미널을 모두 닫고 새로 열어야 합니다. Android Studio도 한 번 재시작하세요. 그래도 안 되면 PATH에 들어간 경로가 C:\src\flutter\bin 인지 다시 확인합니다.

2. cmdline-tools component is missing

SDK Manager → SDK Tools 탭 → Android SDK Command-line Tools (latest)에 체크 후 Apply.

3. Android license status unknown

아래 명령을 다시 실행하고 모든 약관에 y로 동의합니다.

PS> flutter doctor --android-licenses

4. 에뮬레이터가 너무 느리거나 부팅이 안 됩니다.

  • BIOS에서 VT-x(Intel) 또는 SVM(AMD)이 켜져 있는지 확인.
  • Windows 기능에서 Hyper-V 또는 Windows Hypervisor Platform이 켜져 있는지 확인. (Android Studio가 안내하는 옵션을 그대로 켜세요.)
  • System Image를 Google APIs (x86_64) 같은 가벼운 버전으로 바꿔보세요.

5. Gradle build가 JDK 버전 때문에 실패합니다.

아래로 Flutter가 사용하는 JDK를 명시적으로 지정할 수 있습니다.

PS> flutter config --jdk-dir "C:\Program Files\Eclipse Adoptium\jdk-21..."

6. 한글 경로 / 공백 경로 문제

Flutter SDK 경로 또는 프로젝트 경로에 한글, 공백, 특수문자가 있으면 빌드가 깨집니다. 모두 C:\src\... 같은 영문 경로로 옮기세요.

🔍 더 깊이 파보고 싶다면. 공식 트러블슈팅 문서가 가장 정확합니다 → docs.flutter.dev/install/troubleshoot
